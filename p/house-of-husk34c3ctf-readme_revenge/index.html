House of husk&&34c3CTF readme_revenge

Feb 15, 2026

本文参考了下面两篇文章

https://xz.aliyun.com/news/14821

介绍:house of husk是对printf函数内部进行注册自定义格式化字符串函数的指针进行劫持，glibc 2.23-2.35该手法均可用

原理

执行printf函数时，__printf_function_table 是否为空，来判断是否有自定义的格式化字符，如果判定为有的话，则会去执行 __printf_arginfo_table[spec] 处的函数指针，在这期间并没有进行任何地址的合法性检查

那我们如果劫持__printf_function_table非空，劫持 __printf_arginfo_table 使其表中存放的 spec 的位置是 backdoor()，在执行printf函数时就可以劫持程序执行流到backdoor()

利用方法

  • __printf_function_table 和__printf_arginfo_table 分别写为 chunk A 和 chunk B 的地址
  • 设占位符为 α，此时 chunk B 的内容应该为 p64(0) x ord(α-2) + p64(one_gadget)

spec是格式化字符串，比如最后调用的是printf("%X\n",a)那么应该将__printf_arginfo_table[88]的位置写入backdoor(),其中88是ord(‘X’)

  • 该技巧一般和 largebin attack或unsortedbin attack 结合起来
  • 还可以使用relative overwrite
  • 还有一个__printf_va_arg_table 也是可以利用的，但是条件比较苛刻

Relative overwrite

1首先利用largebin attack或unsortedbin attack修改gloabl_max_fast(定义了 fastbin 能够处理的 chunk 最大大小)

fastbinsY数组位于main_area中，当free一个chunk时，会向fastbinsY数组中写入地址

2如果修改了gloabl_max_fast，释放是个大chunk时就会造成数组越界从而向两个table中写入堆地址

/* Register FUNC to be called to format SPEC specifiers.  */
int
__register_printf_specifier (int spec, printf_function converter,
                 printf_arginfo_size_function arginfo)
{
  if (spec < 0 || spec > (int) UCHAR_MAX)   #UCHAR_MAX=0xff
    {
      __set_errno (EINVAL);
      return -1;
    }

  int result = 0;
  __libc_lock_lock (lock);

  if (__printf_function_table == NULL)
    {
      __printf_arginfo_table = (printf_arginfo_size_function **)
    calloc (UCHAR_MAX + 1, sizeof (void *) * 2);
      if (__printf_arginfo_table == NULL)
    {
      result = -1;
      goto out;
    }

      __printf_function_table = (printf_function **)
    (__printf_arginfo_table + UCHAR_MAX + 1);
    }

  __printf_function_table[spec] = converter;
  __printf_arginfo_table[spec] = arginfo;

 out:
  __libc_lock_unlock (lock);

  return result;
}

先做了一个判断 判断spec是否存在ascil码，再判断printf_function_table == NULL 就设定一个calloc来分配两个索引表，并将地址存放到 __printf_function_table和 __printf_arginfo_table中，两个表大小都是0x100可以给0x0-0xff每个字符注册一个函数指针
